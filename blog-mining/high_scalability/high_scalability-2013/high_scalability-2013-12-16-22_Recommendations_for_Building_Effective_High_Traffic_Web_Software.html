5 Hence it is recommended to multiplex application server connections through a multiplexer layer. [sent-55, score-0.461]

8 Consider using a warmup process that is automatically triggered by the PaaS or by your operations team which hits HTTP urls in your application server cluster that warm up the distributed cache. [sent-69, score-0.272]

9 Do this warmup at night or at a time when the customer traffic on those application servers are low. [sent-70, score-0.263]

10 Your client should be able to call ANY application server and have its query answered. [sent-78, score-0.357]

11 The id that maintains state is passed to and from the client for every request and the data that represents a client's session is stored in the database or the distributed cache across multiple requests. [sent-80, score-0.287]

12 Not using session stickiness also allows a rolling restart of your application server cluster thus allowing achieving a 100% uptime. [sent-83, score-0.638]

13 54Embrace all things "reactor"The reactor based pattern has proven itself time and again, a force to reckon for high traffic software systems. [sent-113, score-0.352]

14 A flurry of frameworks have been created that implement the reactor pattern. [sent-114, score-0.266]

15 The places where the reactor can be used are:As a reverse proxy: nGinxAs a application server: node. [sent-115, score-0.401]

16 Implement Call CancelInspired from one of Siddharth Anand's conferences, in a call graph of service calls, first implement timeouts in decreasing number from the top to the bottom. [sent-118, score-0.368]

17 Next for every invocation of a call graph of services, create a UUID and set a flag for the UUID in a distributed cache:UUID: trueIf any service in the call graph times out, set the flag for the UUID to false. [sent-119, score-0.849]

18 Now implement a servlet filter for all services that checks this flag all the time and continues processing only if the flag is true. [sent-120, score-0.473]

19 This allows unnecessary calls from lighting up the call graph during times of high trafficImplement the GC scout protocolAgain, inspired by the same guy - Make all services also expose a TCP port through Netty that returns a empty string. [sent-122, score-0.603]
